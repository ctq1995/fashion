# 图标系统重设计与托盘图标修复设计

## 背景
当前应用已经具备主窗口图标、前端品牌 SVG 和 Tauri bundle 图标资源，但仍存在两个问题：

1. 系统托盘图标未显式设置，导致在 Windows 下可能不显示或显示异常。
2. 现有图标风格偏清爽品牌感，尚未形成“未来极简科幻风”的整套统一视觉系统。

本次设计目标是在不扩大无关范围的前提下，完成托盘图标修复，并重建设备端、打包端、前端品牌端一致的图标系统。

## 目标

### 功能目标
- 修复系统托盘无图标问题。
- 为托盘提供小尺寸高识别度图标。
- 确保主窗口、托盘、打包产物、前端品牌图形保持统一视觉语言。
- 覆盖现有 Tauri bundle 所需各尺寸图标资源，不遗漏已有入口。

### 视觉目标
- 采用“未来极简科幻风”。
- 大尺寸图标具有科技感、时尚感和品牌识别度。
- 小尺寸图标优先保证轮廓识别与对比度，不因细节过多而发糊。
- 保持冷色高亮、深色基底、克制光效，避免厚重霓虹堆叠。

## 非目标
- 不在本次引入新的 logo 动画系统。
- 不扩展启动页、宣传图、网页营销物料。
- 不为不同平台做完全不同的品牌符号，只允许基于同一主徽记做尺寸级简化。

## 设计方向
采用“方案 B：未来极简科幻风”。

### 视觉语言
- 背景主色：深空黑 / 石墨黑。
- 高亮主色：冰青 / 荧光青。
- 辅助色：低比例冷蓝与近白青。
- 图形语言：几何切面、播放语义、音符/节奏结构融合。
- 光效策略：仅保留轻辉光和边缘提亮，不使用厚重描边或复杂噪点。

### 品牌符号构成
主徽记使用单一核心符号，不拆成多套完全不同的图形。主符号融合以下语义：
- 播放三角
- 音乐节奏结构
- 流线轨迹

要求：
- 远看先识别整体轮廓。
- 近看再读到内部节奏与音乐语义。
- 在 16-32 像素尺寸下仍保留清晰主体。

## 尺寸策略
图标系统不采用“一个大 SVG 无差别缩放到所有尺寸”的做法，而是按尺寸分层简化。

### 大尺寸版本
适用范围：
- `src-tauri/app-icon.svg`
- 大尺寸导出 PNG
- 前端品牌展示 SVG

特点：
- 保留背景层次
- 保留少量轨迹与切面
- 有轻微空间感
- 视觉上偏品牌主视觉

### 中尺寸版本
适用范围：
- 64x64
- 48x48
- 32x32 桌面图标显示

特点：
- 减少装饰轨迹
- 保留主体徽记与对比层
- 优先图形识别，不强调细微渐变

### 小尺寸与托盘版本
适用范围：
- 16x16
- 20x20
- 24x24
- Windows 托盘

特点：
- 极简轮廓
- 可采用单色或准单色版本
- 去除内部细小装饰和复杂高光
- 优先在暗色任务栏下可见

## 覆盖范围
本次实现必须覆盖以下资源与入口：

### 源图与品牌资源
- `src-tauri/app-icon.svg`
- `src/assets/fashion-brand.svg`
- 如有必要，新增托盘专用源图（例如 `src-tauri/tray-icon.svg`）

### Tauri bundle 入口
- `src-tauri/tauri.conf.json` 中声明的打包图标
- `src-tauri/icons/32x32.png`
- `src-tauri/icons/128x128.png`
- `src-tauri/icons/128x128@2x.png`
- `src-tauri/icons/icon.ico`
- `src-tauri/icons/icon.icns`

### 已存在的移动端图标资源链路
- `src-tauri/icons/android/**`
- `src-tauri/icons/ios/**`

要求：
- 不遗漏当前仓库内已存在的图标资源入口。
- 若采用统一源图再生成，生成结果需覆盖现有资源链路。

## 托盘图标修复方案
当前系统托盘创建位于 `src-tauri/src/lib.rs` 中的 `build_system_tray(...)`。问题在于：
- 当前托盘创建只设置 menu 与事件
- 未显式设置 tray icon

修复设计：
1. 准备托盘适配图标资源。
2. 在 `TrayIconBuilder` 构建时显式设置图标。
3. 优先确保 Windows 托盘环境下图标可见且清晰。
4. 不依赖“主窗口图标自动继承到托盘”这一不稳定前提。

## 生成与维护策略
为了避免后续维护成本过高，本次应采用“源图 + 生成产物”的方式：

1. 重做统一主 SVG 源图。
2. 如需要，为托盘维护一个小尺寸专用 SVG 源图。
3. 通过工具链生成 PNG / ICO / ICNS / 平台图标资源。
4. 保留可重复生成的流程，避免手工逐个编辑所有位图资源。

要求：
- 资源生成流程应可重复执行。
- 若项目已有 Tauri 图标生成能力，则优先复用。
- 若需额外脚本，脚本应只服务本次图标生成，不扩展无关资产流水线。

## 验收标准
完成后需满足以下标准：

### 功能验收
- 系统托盘显示图标。
- 左键托盘图标交互不受影响。
- 主窗口图标、打包图标、托盘图标资源链路完整。

### 视觉验收
- 大图具备未来极简科幻风格。
- 小图不糊、不脏、不丢失主体。
- 前端品牌 SVG 与打包图标视觉统一。
- 托盘图标在深色任务栏中具有足够对比度。

### 工程验收
- `tauri.conf.json` 图标声明仍有效。
- Rust 侧托盘图标设置编译通过。
- 构建流程可正常产出应用图标资源。

## 实施边界
本次只处理与图标系统和托盘图标相关的改动：
- 图标源文件
- 生成后的图标资源
- Tauri 托盘设置
- 必要的生成脚本或命令

不顺带修改：
- 播放器业务逻辑
- 迷你播放器布局
- 其他无关 UI 组件
