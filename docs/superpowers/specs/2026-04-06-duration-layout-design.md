# 歌曲总时长统一显示设计

## 背景
当前项目中的歌曲列表面板在“总时长”展示上不统一：

- 本地音乐模块已经显示时长，但与操作区之间仍然偏松，不够紧凑
- 我的喜欢、搜索结果、播放列表都没有稳定显示歌曲总时长
- 最近播放显示的是播放时间和停留进度，但缺少歌曲总时长
- 不同面板的列表信息密度不一致，导致横向节奏和层级不统一

用户希望两件事同时成立：
- 音乐时间位置重新设计，更紧凑
- 其他地方也补上歌曲总时长，且尽量全局统一

## 目标

### 主要目标
- 在所有核心歌曲列表中统一显示歌曲总时长
- 总时长使用稳定的右侧字段展示，不被长文本挤掉
- 本地音乐模块的时长与按钮区排布更紧凑
- 保持各面板的主要操作逻辑不变
- 提升不同面板之间的列表一致性

### 非目标
- 不修改播放逻辑
- 不新增排序、筛选或搜索条件
- 不改动历史记录数据结构
- 不引入新的全局 store
- 不重构整个面板体系

## 设计结论
采用方案 B：总时长作为独立右侧字段统一展示。

核心原则：
- 歌曲标题和副标题保留在左侧主信息区
- 总时长放在右侧稳定区域
- 操作按钮继续靠右收口
- 避免把总时长塞进副标题末尾，防止被长文本省略

## 适用范围
本次统一覆盖以下面板：

- `src/components/LocalLibraryPanel.vue`
- `src/components/FavoritesPanel.vue`
- `src/components/SearchPanel.vue`
- `src/components/PlaylistPanel.vue`
- `src/components/HistoryPanel.vue`

## 结构设计

### 1. 本地音乐
当前本地音乐已经具备右侧总时长，本次只做位置精修：

- 保持总时长位于右侧
- 缩小总时长与按钮区之间的水平间距
- 保持右侧信息区更紧凑
- 保持窄宽度下仍可换到下一行，但优先保证时长清晰可见

### 2. 我的喜欢
从当前“左侧信息 + 右侧按钮”改成：

- 左侧：序号、封面、歌名、歌手 / 专辑
- 右侧：总时长、操作按钮

总时长放在按钮左侧，使用与本地音乐相近的字号和数字对齐方式。

### 3. 搜索结果
当前搜索结果右侧已有音源标签与操作按钮，本次调整为：

- 左侧：序号、封面、歌名、歌手 / 专辑
- 右侧：音源标签、总时长、操作按钮

要求：
- 总时长要独立可见
- 不被来源标签或按钮挤掉
- 长标题情况下，优先压缩左侧文本区

### 4. 播放列表
当前播放列表右侧只有操作按钮，本次调整为：

- 左侧：封面、歌名、歌手
- 右侧：总时长、操作按钮

若当前歌曲处于播放中，保留“正在播放”标签，不影响总时长显示。

### 5. 最近播放
最近播放是特殊面板，右侧已有两类信息：
- 播放时间
- 播放进度/停留位置

本次调整为右侧双层信息区：
- 第一行：播放时间
- 第二行：停留位置 + 总时长

呈现形式建议为：
- 已播完：`已播完 · 03:42`
- 未播完：`停在 01:18 · 03:42`

这样既保留历史语义，也补上歌曲总时长。

## 视觉设计

### 信息优先级
统一优先级如下：
1. 歌名
2. 歌手 / 专辑
3. 总时长 / 来源 / 历史状态
4. 操作按钮

### 时长样式
- 使用较小但稳定的数字字号
- 开启等宽数字显示，避免时间跳动感
- 颜色低于标题、高于弱提示信息
- 宽度保持稳定，避免不同歌曲时长造成布局抖动

### 紧凑性要求
- 时长与按钮区间距比当前本地音乐更小
- 时长不做胶囊，不增加额外装饰
- 所有面板优先采用“文本区压缩、右侧区稳定”的布局策略

## 交互设计
- 总时长仅用于展示，不可点击
- 原有双击播放、播放按钮、收藏、下载、删除、加入队列等行为保持不变
- 不改变“正在播放”“音源标签”“下载按钮”等现有含义

## 实现方案
主要改动文件：
- `src/components/LocalLibraryPanel.vue`
- `src/components/FavoritesPanel.vue`
- `src/components/SearchPanel.vue`
- `src/components/PlaylistPanel.vue`
- `src/components/HistoryPanel.vue`

可复用实现内容：
- 各面板内本地 `formatDuration` / `fmtTime` 逻辑统一用于总时长展示
- 通过样式层的 `row-side` / `action-row` / `drawer-actions-row` 等区域调整实现统一收口

计划改动：
1. 本地音乐调整时长与按钮区间距
2. 我的喜欢新增时长显示
3. 搜索结果新增时长显示并重排右侧区域
4. 播放列表新增时长显示
5. 最近播放补充总时长显示并重排右侧两行信息
6. 为新增的时间展示补最小测试覆盖

## 风险与控制

### 风险
- 搜索结果右侧已有来源标签和多个按钮，新增时长后容易再次拥挤
- 播放列表宽度较窄，新增时长后可能压缩标题空间
- 最近播放右侧信息较多，若组织不好会显得过密

### 控制措施
- 一律优先压缩左侧文本区，不牺牲右侧时间可见性
- 在窄宽度下允许右侧信息区换行，但保持时间优先可见
- 最近播放采用两行右侧信息，避免一行塞满
- 保持改动局限于面板内部模板与样式，不扩散到 store 结构

## 验证方案

### 自动验证
- `npm test`
- `npm run build`

### 手动验证重点
- 本地音乐时长位置比当前更紧凑
- 我的喜欢每首歌都显示总时长
- 搜索结果每首歌都显示总时长，且不与来源标签冲突
- 播放列表每首歌都显示总时长
- 最近播放同时显示播放时间、进度状态和总时长
- 长歌名下时长依然可见
- 窄宽度下不出现明显重叠或挤压

## 完成标准
满足以下条件即可认为本次设计落地完成：

- 五个核心列表面板都能稳定显示歌曲总时长
- 本地音乐时长位置比当前更紧凑
- 右侧时间字段不会被长文本轻易挤掉
- 原有操作行为不变
- 测试与构建通过
