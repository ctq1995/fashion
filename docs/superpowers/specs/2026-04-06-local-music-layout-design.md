# 本地音乐模块布局统一设计

## 背景
当前本地音乐模块已经具备文件夹添加、扫描、展示、播放、收藏、加入队列和移除曲目的能力，但界面结构仍然存在明显问题：

- 顶部文件夹区域直接展示本地路径，信息暴露过多
- 列表每行同时放入歌名、歌手/专辑、时长、文件名、完整路径和操作按钮，横向信息密度过高
- 当面板宽度不足时，文本区与按钮区容易相互挤压，出现重叠、拥堵和层级混乱
- 当前视觉组织方式与项目中其他内容面板不够统一

本次调整只处理本地音乐模块的布局与信息展示，不改变本地音乐的核心功能行为。

## 目标

### 主要目标
- 本地音乐模块与其他模块保持更统一的整体风格
- 不再显示任何本地路径信息
- 解决列表项在常见宽度下的重叠、挤压和拥堵问题
- 保留当前播放、收藏、加入队列和移除能力
- 让歌曲信息层级更清楚，操作区更稳定

### 非目标
- 不改动本地音乐扫描逻辑
- 不改动本地音乐播放逻辑
- 不增加新的筛选、排序或搜索功能
- 不改动本地音乐数据结构
- 不重构整个页面导航或其他模块布局

## 设计结论
采用方案 B：按现有全局列表风格收齐。

核心原则如下：
- 删除所有路径展示，包括顶部文件夹路径和歌曲行中的完整路径
- 弱化本地音乐顶部展示区，只保留标题、数量、操作按钮与扫描反馈
- 列表回归稳定的内容区 + 操作区结构
- 每行只保留核心播放信息，不再堆叠次要文件信息

## 结构设计

### 顶部区域
保留顶部区域，但从“信息展示区”收敛为“模块标题 + 操作区”：

- 保留“本地音乐”标题
- 保留数量标记
- 保留“添加文件夹”和“扫描曲库”按钮
- 保留扫描结果提示和错误提示
- 移除当前顶部 hero 曲目信息中的路径相关延展感
- 移除文件夹路径横向 strip

这样顶部仍然保留模块身份，但不会因为路径 chip 导致信息杂乱。

### 列表区域
每条本地音乐改成更稳定的双区结构：

- 左侧：序号、封面、歌名、歌手 / 专辑
- 右侧：时长、操作按钮

从现有行结构中移除以下字段：
- `item.record.fileName`
- `item.record.path`

保留以下字段：
- `item.track.name`
- `item.track.artist`
- `item.track.album`
- `item.record.durationSec`
- 当前播放状态标签
- 收藏 / 加入队列 / 播放 / 删除按钮

## 视觉设计

### 信息层级
- 歌名作为第一层信息，优先保证单行省略稳定显示
- 歌手与专辑合并为第二层信息，继续单行省略
- 时长移到右侧稳定区域，不再混在元信息中
- 删除文件名和路径后，每行信息量保持在可快速扫读的范围内

### 布局一致性
- 顶部与列表区域继续沿用现有面板体系的圆角、边框、间距风格
- 列表项以更明确的左右分区组织，减少本地模块特有的“多段信息堆叠”
- 操作按钮区固定收口，不再参与挤压文本区

## 交互设计
- 双击行播放保持不变
- 收藏、加入队列、播放、删除按钮行为保持不变
- 当前播放中的歌曲仍显示“播放中”标签
- 顶部按钮行为保持不变

本次不引入新的交互，也不改变已有操作路径。

## 实现方案
主要改动文件：
- `src/components/LocalLibraryPanel.vue`

计划改动：
1. 删除顶部文件夹路径展示区
2. 精简顶部文案区，减少不必要的曲目展示负担
3. 删除行内 `row-file` 与 `row-path` 的模板输出
4. 调整 `local-row`、`row-content`、`row-detail`、`row-actions` 的布局样式
5. 将时长从拥挤的信息区中抽离到右侧稳定区域
6. 增加必要的最小宽度、溢出和省略控制，避免重叠

## 风险与控制

### 风险
- 删除路径后，用户对来源位置的可见性会降低
- 行结构调整后，如果右侧按钮区宽度控制不好，仍可能压缩标题区域
- 顶部区域若收得过度，可能显得信息不足

### 控制措施
- 保留顶部标题、数量和扫描反馈，避免界面过空
- 保证左侧文本区 `min-width: 0` 并持续使用单行省略
- 固定右侧时长与按钮区的布局边界，避免挤压扩散
- 保持改动范围只在本地音乐模块内部

## 验证方案

### 自动验证
- `npm run build`
- 与本地音乐相关的现有测试

### 手动验证重点
- 顶部不再显示任何路径信息
- 列表中不再显示文件名和完整路径
- 常见宽度下，歌名、副标题、时长和操作按钮不重叠
- 长歌名时按钮区不会被顶乱
- 双击播放、按钮点击、删除、收藏功能正常

## 完成标准
满足以下条件即可认为本次调整完成：

- 本地音乐模块不再显示任何路径信息
- 列表布局在常见窗口宽度下稳定，无重叠和明显挤压
- 本地音乐模块视觉结构与其他模块更统一
- 现有功能行为保持不变
- 构建与相关测试通过
