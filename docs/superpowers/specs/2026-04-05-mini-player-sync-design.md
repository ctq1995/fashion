# 迷你播放器同步与入口设计

## 背景
当前迷你播放器已经具备独立窗口与基础播放控制能力，但在使用体验上仍有三个明显问题：

- 迷你播放器入口主要不在全局顶部区域，切换路径不够直接
- 主播放器栏和迷你播放器可能在不同场景下形成重复感，缺少明确的互斥模式
- 迷你播放器缺少播放顺序控制，和主播放器的核心控制能力不对齐

本次工作目标是把迷你播放器收敛为桌面端的另一种播放器展示模式，而不是并列存在的第二套播放器界面。

## 目标

### 主要目标
- 将迷你播放器切换入口放到全局右上角
- 打开迷你播放器时隐藏主窗口底部主播放器栏
- 关闭迷你播放器时恢复主播放器栏
- 迷你播放器与主播放器共享同一份播放状态与控制动作
- 迷你播放器补齐播放顺序控制能力

### 非目标
- 不改变移动端播放器行为
- 不隐藏或最小化主窗口本身
- 不新增第二套播放器状态存储
- 不重构 Tauri 托盘逻辑
- 不扩展成多窗口多实例播放器模式

## 状态同步设计
迷你播放器和主播放器不维护两套状态，统一复用 `src/stores/player.ts` 中已有的播放器 store。

共享状态包括：
- 当前曲目
- 播放 / 暂停状态
- 当前时间
- 总时长
- 播放顺序
- 队列位置
- 迷你播放器显隐状态

共享动作包括：
- `togglePlay()`
- `playPrev()`
- `playNext()`
- `seek()`
- `togglePlayMode()`
- `setMiniPlayerVisible()`

这样主播放器与迷你播放器的同步由同源状态天然保证，不额外增加前端状态桥接层。

## 入口设计
迷你播放器切换入口从设置页下沉为次入口，从全局顶部栏升格为主入口。

桌面端主入口放在：
- `src/components/TitleBar.vue`

入口行为：
- 当迷你播放器关闭时，点击按钮打开迷你播放器
- 当迷你播放器打开时，点击按钮关闭迷你播放器并恢复主播放器栏
- 按钮文案与 title 应根据当前状态切换，避免语义模糊

设置页中原有迷你播放器入口如果保留，不再作为主操作路径，只做补充入口或说明，避免主入口和次入口行为不一致。

## 互斥显示设计
桌面端播放器展示采用“互斥切换”模式，而不是“同时存在”模式。

规则如下：
- `player.showMiniPlayer === true` 时：
  - 主窗口底部主播放器栏不渲染
  - 迷你播放器窗口可见
- `player.showMiniPlayer === false` 时：
  - 主窗口底部主播放器栏正常渲染
  - 迷你播放器窗口隐藏

主窗口本身继续保留，用户仍可以浏览搜索、歌单、歌词页和设置页。

移动端不接入这套窗口切换逻辑，避免移动端布局被桌面窗口模式污染。

## 迷你播放器功能补齐
迷你播放器补齐与主播放器一致的播放顺序控制能力。

支持模式：
- 顺序播放
- 随机播放
- 单曲循环

实现方式：
- 直接读取 `player.playMode`
- 直接调用 `player.togglePlayMode()`
- 复用现有模式文案与图标语义，不再定义新枚举

这样任一端切换播放顺序，另一端都会实时同步。

## 文件变更范围

### 前端
- `src/components/TitleBar.vue`
  - 增加桌面端全局迷你播放器切换按钮
- `src/App.vue`
  - 控制主播放器栏的互斥显示
  - 统一迷你播放器打开/关闭逻辑
- `src/MiniPlayerApp.vue`
  - 增加播放顺序按钮
  - 与主播放器共享播放顺序状态
- `src/components/SettingsPanel.vue`
  - 清理或弱化重复迷你播放器入口，避免入口冲突

### Rust
- 原则上不改 `src-tauri/src/lib.rs`
- 只有在实现过程中发现窗口开关语义不满足互斥模式时，才做最小调整

## 风险与控制

### 风险
- 主入口和设置页入口同时存在时，用户认知可能混乱
- `showMiniPlayer` 与窗口真实显示状态可能短暂不同步
- 桌面端互斥显示处理不当时，可能出现主播放器栏与迷你播放器同时可见

### 控制措施
- 统一通过 `setMiniPlayerVisible()` 驱动 UI 状态
- 迷你播放器显示与关闭继续复用现有 ready/closed 事件
- `App.vue` 只保留一套桌面端主播放器栏显示条件
- 设置页不再承担主入口职责

### 自动验证
- `npm test -- src/stores/ui.test.ts src/stores/localLibrary.test.ts src/stores/library.test.ts`
- `npm run build`
- `cargo check --manifest-path "E:/Polaris/music/src-tauri/Cargo.toml"`

### 手动验证重点
- 顶部右上角按钮可以打开迷你播放器
- 打开迷你播放器时，主播放器栏立即隐藏
- 关闭迷你播放器时，主播放器栏立即恢复
- 主播放器与迷你播放器的播放/暂停、上一首、下一首保持同步
- 迷你播放器切换播放顺序后，主播放器顺序状态同步更新
- 任意时刻桌面端不会同时看到主播放器栏和迷你播放器

## 完成标准
满足以下条件即可认为本次功能完成：

- 迷你播放器入口已迁移到全局右上角
- 桌面端主播放器栏与迷你播放器实现互斥切换
- 迷你播放器具备播放顺序控制能力
- 主迷你播放器基于同一个 store 同步工作
- 测试、构建、Rust 编译检查全部通过
